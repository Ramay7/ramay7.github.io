    CF #378 C (2016/11/1) 模拟漏判初始条件
    题目链接：Here
    题意：有$n\leq 500$个怪兽，给出每个怪兽的体重($\leq 10^6$)，当一个怪兽的体重严格大于另一个怪兽的体重时，体重大的可以吃掉体重小的，新的怪兽的体重是原来两个怪兽体重之和。并且一个怪兽（除了首尾两个）只能吃左右相邻的两个（如果体重条件符合的话，首只可以吃第二个，尾只可以吃倒数第二个）。每次一个怪兽被吃掉后，这些怪兽会从首至尾从1开始重新编号。现在给出经过$n-K$次操作后，剩下的$K$个怪兽的体重，问能否从初始序列得到这$K$个怪兽的体重？如果可以，输出任意一种合法的顺序。

$x L$表示位置下标为$x$的吃掉它左边的怪兽，$x R$表示位置下标为$x$的吃掉它右边的怪兽。

因为每个怪兽只能吃相邻的怪兽，所以最终剩下的$K$个怪兽的每一个肯定都是原始序列中的连续一段形成的。那么只需要判断连续的一段和能否和这$K$个一一对应即可。找到了每一段的起始位置，还要找到这一段的第一个最大值的位置，因为一定是从最大值开始吃的。所以如果这一段所有数字都相同的话（相当于都是最大值），那么肯定也是不合法的。

如果最大值是这一段的第一个位置的话，要往后找到第一个最大值的位置，使得它后面位置的体重比它小，然后先吃后面一段的，再吃前面一段。如果最大值不是这一段的第一个位置的话，那就先吃前面一段，再吃后面一段，模拟一下，注意编号变化。

FST原因

没有判断前后两个序列体重之和是否相等，如果体重不相等的话肯定是不合法的！例如：

5
1 2 3 4 5
1
10

CF #379C (2016/11/16) 分类讨论少分类

题目链接：Here
题意：
需要生产$n\leq 2e9$件物品，每件物品的生产时间是$x\leq 2e9$，有两种加速操作：

  • 第一种操作有$m\leq 2e5$个选择，耗费$b[i]\leq 2e9$的代价可以使得每件物品的单位生产时间改变为$a[i] < x$
  • 第二种操作有$K\leq 2e5$个选择，耗费$d[i]\leq 2e9$的代价可以立即生产$1\leq c[i]\leq n$件商品，并且保证满足：$c[i] \leq c[j]\quad d[i] \leq d[j]\quad (i < j)$，两个关系同时成立。

现在第一种操作和第二种操作至多都只能从中选择1个，在不超过代价$S\leq 2e9$的条件下，求生产$n$件物品的最小代价？

分析：
最优解一定从以下四种情况中产生：

  • 第一种选择和第二种选择都不选
  • 只选择第一种方案中的某一个选择
  • 只选择第二种方案中的某一个选择
  • 第一种方案和第二种方案各选择一个

需要注意代价和不超过$S$,同时注意到$c[i] \leq c[j]\quad d[i] \leq d[j]\quad (i < j)$，可以把第一种方案的所有选择按照$b[i]$从小到大排序，然后在第四种情况时从$i:K\rightarrow 1$枚举第二种方案中的选择，第一种方案中选择满足$b[st]\leq S - d[i]$中的最小的$a[st]$，类似尺取的思路。

FST原因：

在第二种情况中， 即只选择第二种方案中的某一个选择时只考虑到：当$d[i]\leq S$&&$c[i]\geq n$时，$ans=0$，忘记了当$d[i]\leq S$ &&$c[i] < n$时，$ans=\min (ans, 1LL * (n - c[i])*x)$，实际上题目中已经说了$c[i] < n$，智障了。

时间复杂度：$O(m\log m + K)$

CF #383B (2016/12/7) 忘记处理边界

题目链接：Here
题意：
给$n\leq 10^5$个数和$x\in [0,10^5]$，求$A[i]$ xor $[A[j]=x(1\leq i < j \leq n,A[i]\in [1,10^5])$的数字对数。

分析：
如果$a$ xor $b = x$，那么$a$ xor $x=b$。因此对于每个$A[i]$，我们只需要统计$A[i]$ xor $x$出现的次数，需要注意$A[i]$ xor $x$可能大于$10^5$，这种情况应该直接忽略，防止RE。最后再把答案除以2。

但是这样正确吗？

FST原因

没有考虑数字重复和$x=0$的情况。正确的做法是，逐个统计：统计在它之前出现的数字中$A[i]$ xor $x$出现的次数，这样子也就不用去重了。
